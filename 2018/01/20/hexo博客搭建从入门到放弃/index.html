记录一下我在搭建这个博客当中遇到的技术问题吧

我的博客使用了Hexo 。Hexo 是一个轻量级的博客框架，它可以高度定制，不仅插件齐全，而且主题繁多，可以满足很多个性化的需求。麻雀虽小，五脏俱全，相比于功能齐全但体积庞大的 WordPress，Hexo 有其独特的优势。

安装 Hexo

安装过程可以参考 ： hexo的搭建、设置与部分优化
基本就是按照里面的步骤一步一步来就可以了，这里我没有遇到什么问题

部署 Hexo 到 Github 上

这里遇到了最多的问题，下面我依次进行分析。

部署执行失败

我的电脑有 .ssh 密钥（以前为某个项目生成过），也将公钥添加到了项目 Setting 里的 Deploy key 里面，但无论怎样部署 Github 都提示拒绝访问 。

经过研究发现，问题可能出在我的密钥上，因为以前的某个项目已经使用了这一密钥，可能发生了冲突，因此重新生成密钥，但是部署密钥后仍然是拒绝访问，看起来问题不在这上面。

在设置 Git 的时候，我想起来 Git 的 origin 好像还是之前的项目的地址，没有改过来。确实如此，便修改了origin后，可结果还是拒绝访问。这时候我注意到 Setting 里的 Deploy key 页面里面有这个东西：

勾选之后，部署就成功了，可以在项目的 master 目录里看到页面资源了，问题解决。

部署成功后一直 404

这里困扰了我非常久。我参照： Hexo常见问题解决方案 里的方法，对各种设置进行了核对，并没有发现问题所在。参考了其他人了项目 master 目录结构后，我也没有发现特别大的区别。到处找解决方案也无果。这就很尴尬了，难不成就我自己一个人遇上了这个问题？

经过细致的排查，我终于发现了问题：我之前有两个项目都使用了 Github Pages，而且 Github Pages 的域名刚好是我的博客的域名，很有可能发生了占用冲突。因此将那两个项目中的 Github Pages 手动关闭，博客就能正常访问了。

个性化 Hexo 博客

博客跑是能跑起来了，但还有很多地方可以提升。

主题更换与选择

博客默认的主题是 landscape ，这个不太好看，我希望能换个主题，最好能有种简洁明快的感觉。于是很自然的，我找到了 Hexo 主题官网 ：

官网的主题非常多，看了一圈，我比较喜欢 NexT 和 Material 的风格。
NexT 很火，很多人用，但是看久了就感觉太单薄了。所以我选择了 Material 。

Material 安装比较简单，就是把文件解压到 themes 文件夹下，然后按要求改 _config.yml 就差不多了。
具体方法可以参考 Material 的官方指南 。

主题配置修改

参考 Material 的官方指南 即可。

然后我遇到了中文编码问题，主要表现为：根目录下的 _config.yml 按之前 hexo的搭建、设置与部分优化 中的要求，将 language 值设置成了 zh-Hans，但整个网站界面的语言都不是汉语，而是其他语言：

经过分析，问题出在 Material 主题使用的语言文件上。
由于 Material 简体中文界面的语言文件名为 zh-CN.yml，而非 zh-Hans.yml，

因此当在 _config.yml 里将 language 设置成 zh-Hans 后，
在 Hexo 打包文件的过程中会它找不到 zh-Hans.yml，于是默认读取了第一个阿拉伯文的语言文件。

因此，将 language 改成 zh-CN，即可让界面正常显示中文：

图片上传及显示

这个问题还是挺麻烦的，我也纠结了很久。

在博客中显示自己的图片有两种方法：

1.上传到图床，然后通过外链引用图片；
2.从本地上传，随博客一起发布；

两种方法的优缺点都很明显：

前者不占本地 和 GitHub 的空间，一般都带有 CDN 加速，引用很方便，但是每次都要事先把图片上传到云端；
后者操作简单，配置不难，但占本地 和 GitHub 的空间，配置不当会出现引用问题。

经过慎重考虑，我还是选择了从本地上传的方法，因为每次都要把图片上传到图床上太麻烦了。
话不多说，开始动手。
